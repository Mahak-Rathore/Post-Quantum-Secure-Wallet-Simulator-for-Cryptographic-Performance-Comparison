POST-QUANTUM WALLET SIMULATOR - CHANGELOG & ARCHITECTURAL OVERVIEW
========================================================================
This document summarizes all modifications and improvements made to the original directory since the overhaul began.

1. FULL PROJECT RESTRUCTURING AND DECOUPLING
------------------------------------------------------------------------
- Monolithic structures (if any) have been separated into a client-server model.
- Created `frontend/` directory for the React-based client web application.
- Created `backend/` directory for the Node.js Express application handling core logic.

2. FRONTEND (UI/UX) OVERHAUL
------------------------------------------------------------------------
- **React Component Architecture**: Refactored the UI into a modular, component-based React system located in `frontend/src/components` (e.g., Dashboard, Key Generation, Quantum Adversary, Ledger Streams).
- **Styling**: Integrated TailwindCSS (`frontend/tailwind.config.js`) to establish a futuristic, dark-mode aesthetic with neon accents explicitly requested for an academic presentation.
- **Responsiveness**: Rewrote CSS logic (accessible in `frontend/src/index.css`) to ensure dynamic alignment and a premium UI experience. Waitings, layout flexing, and micro-animations have been added to improve user engagement.
- **Information Density**: Cryptographic data, including full-length key structures (Classical RSA/ECC vs Post-Quantum Lattice), payload signatures, and benchmarking latencies are visually grouped for clear comparison.

3. BACKEND (SERVER) ARCHITECTURE
------------------------------------------------------------------------
- **Node.js Express Server**: Standardized a REST API located in `backend/server.js` running with specific modularized routing.
- **RESTful Routes (`backend/routes/`)**: Created distinct API endpoints for different workflows:
   - `walletRoutes.js`: Handles generation of Classical and Lattice-based PKI keypairs.
   - `transactionRoutes.js`: Responsible for creating transfers and generating cryptographic signatures.
   - `experimentRoutes.js`: Manages the global benchmarking warming-up functions.
   - `benchmarkRoutes.js`: Fetches comparative metrics related to latencies, key sizes, and payload structures.
- **Services (`backend/services/`)**: Isolated robust crypto logic for RSA/ECC, Post-Quantum operations, and the "Quantum Adversary" Shor's algorithm simulator.
- **Data Persistence**: Configured JSON-based local data storage (e.g., `backend/data/transactions.json`) to persist network events and ledger history for evaluation purposes.

4. MISCELLANEOUS FILES
------------------------------------------------------------------------
- `walkthrough.txt`: Created an exhaustive, academic step-by-step demo script allowing a presenter to sequentially show Wallet initialization, Quantum attacking, executing transfers, and analyzing cryptographic overhead data.

========================================================================
All code now focuses heavily on presenting a clean visual comparison and maintaining separated programmatic logic for secure crypto simulations.
